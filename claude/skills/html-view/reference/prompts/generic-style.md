# generic-style: フォールバック (シンプル markdown HTML)

## 目的

review / plan / fix-plan / research のいずれにも該当しない Markdown ドキュメント (雑メモ / ADR ドラフト / 議事録など) を、`style-guide.md` 準拠の標準的な markdown HTML に変換する。装飾は最小限、可読性を優先。

## 期待される構造

```
<header>
  <h1>{title (入力 MD の最初の # 行、または basename)}</h1>
  <p class="generated-at">generated at {ISO 8601 timestamp}</p>
</header>

<main>
  <!-- 入力 MD の h2/h3/段落/リスト/コードブロック/テーブル/blockquote をそのまま HTML 化 -->
</main>

<footer>
  <p class="muted">generated by /html-view (style: generic)</p>
</footer>
```

## デザイン指示

- **style-guide.md 準拠** を厳守:
  - CSS 変数: `--bg`, `--card-bg`, `--border`, `--text`, `--muted`, `--code-bg`, `--link` を `:root` で定義
  - フォントスタック: system-ui 系 + 日本語フォント明示
  - `max-width: 880px; margin: 0 auto; padding: 24px;` のシンプルレイアウト
- **見出し階層**:
  - `h1`: `font-size: 1.9em; border-bottom: 2px solid var(--border); padding-bottom: 8px;`
  - `h2`: `font-size: 1.5em; border-bottom: 1px solid var(--border); padding-bottom: 6px; margin-top: 2em;`
  - `h3`: `font-size: 1.2em;`
  - `h4`: `font-size: 1.05em; color: var(--muted);`
- **コード**:
  - インライン `<code>`: `background: var(--code-bg); padding: 2px 6px; border-radius: 4px; font-family: monospace;`
  - `<pre>`: `background: var(--code-bg); padding: 12px 16px; border-radius: 6px; overflow-x: auto;`
- **テーブル**: 標準 border + ヘッダ背景 `--code-bg`
- **blockquote**: `border-left: 4px solid var(--border); color: var(--muted); padding: 8px 16px; background: var(--card-bg);`
- **リスト**: 標準スタイル、`padding-left: 1.6em;`
- **`@media print` 対応必須** (style-guide 規約)

## 参考にすべき example

なし (シンプルなので example 不要)。`style-guide.md` の規約だけで十分構築可能。

## 必須要素チェックリスト (grep パターン)

```
CSS 変数定義:    --bg:\s*#  (style-guide のパレットを採用していること)
タイトル:        <h1>
print 対応:      @media\s+print
フォント:        font-family:.*?(system-ui|-apple-system)
viewport:        <meta name="viewport"
lang 属性:       <html lang="ja"
```

少なくとも 6 パターンすべてが grep で検出されること。装飾の少ない style だが、style-guide 準拠と単体完結原則は厳守。
