# 統合テスト結果報告テンプレート

## status 4区分（緑で丸めない）

- ✅ **実機live**: UI操作→実際のnetworkリクエストを実捕捉
- ✅ **実API再現**: UI駆動不能なため、該当処理を担う関数/APIリクエストを同じ手順で直接実行して検証
- ✅ **endpoint共有**: 別の実機live確認済み経路と同一endpoint・同一コア関数であることをsourceで確認
- ⚠️ **データ未到達**: 環境にテストに必要なデータが存在しない。core/endpointの妥当性はsource確認のみ。データ作成が多段ウィザード等で脆く副作用を伴う場合、無理に作成せずこの区分に留める

## 報告構成

① サマリ表（status区分×件数）
② 観点別全経路結果
③ 経路別テーブル（URL / 発行method+status / 実測status / トースト文言 等）
④ 実測ログ抜粋
⑤ 発見・注意（経路固有の制約、判定チェックリストで切り分けた環境要因の記録を含む）
⑥ 参照実装（該当コアの実装ファイルパス）
