# Notion ADR 検索スキル

Notion に記録された ADR（Architecture Decision Record）を検索・取得し、過去の技術判断の背景や決定事項を確認します。

## When to Use

- 技術調査で過去の設計判断の背景を確認するとき
- 実装方針を検討する際に、以前の意思決定を参照するとき
- 「なぜこの設計にしたか？」「以前試して失敗したアプローチは？」を調べるとき
- 新しい技術選定で、過去の採用・棄却の理由を確認するとき

## 推奨アプローチ（効率的）

### 方法A: セマンティック検索（1回のAPI呼び出し）

`user-Notion:notion-search` でADRデータベースを対象にキーワード検索：

```
user-Notion:notion-search
引数: {
  "query": "<調査対象の技術・設計トピック>",
  "data_source_url": "collection://e2bf3bd4-5237-400d-b5f2-307508d2a5f2"
}
```

**メリット**: 1回のMCP承認で関連するADRを複数取得可能
**注意**: セマンティック検索のため、完全一致ではなく関連性ベースの結果が返る

**検索キーワードのコツ**:
- 技術名を含める（例: `認証 JWT OAuth`）
- 問題領域を含める（例: `パフォーマンス キャッシュ戦略`）
- 採用状況で絞り込む場合は含める（例: `承認済み API設計`）

### 方法B: フィルター済みビューをクエリ（正確なフィルタリング）

Notionで特定条件のフィルター済みビューを事前作成し、そのURLでクエリ：

```
user-Notion:notion-query-database-view
引数: { "view_url": "https://www.notion.so/e2bf3bd45237400db5f2307508d2a5f2?v=d744d148d89042689c066123393a3a3b" }
```

**メリット**: Notion側のフィルタ条件（採用状況、日付範囲等）が正確に適用される

## 代替アプローチ（個別取得）

検索で十分な結果が得られない場合のフォールバック：

```
ADR取得進捗：
- [ ] Step 1: ADRデータベースのビューURLでクエリし、一覧を取得
- [ ] Step 2: 関連するADRのURLを特定
- [ ] Step 3: 各ADRページを個別取得（notion-fetch）
- [ ] Step 4: 結果をサマリー出力
```

### Step 3: 個別ADR取得

```
user-Notion:notion-fetch
引数: { "id": "<ADRページのURL>" }
```

レスポンスからタイトル、採用状況、文脈、決定事項を抽出する。

## 結果出力フォーマット

```markdown
## 関連ADR一覧（N件）

| タイトル | 採用状況 | 作成日 | URL |
|---------|---------|--------|-----|
| 【ADR】認証方式の選定【2025-06】 | 承認済み | 2025-06-15 | URL |
| 【ADR】キャッシュ戦略【2025-03】 | 棄却 | 2025-03-10 | URL |

### 詳細: 【ADR】認証方式の選定【2025-06】

**文脈**: ...
**決定事項**: ...
**結果**: ...
```

## 主要プロパティ

| プロパティ | 説明 |
|-----------|------|
| `タイトル` | ADRタイトル（【ADR】タイトル【YYYY-MM】形式） |
| `採用状況` | 提案済み / 承認済み / 棄却 / 非推奨 / 置き換え済み |
| `公開` | 公開ステータス（Yes / No） |
| `Collaborator` | 作成者 |
| `作成日` | ADR作成日 |
| `最終更新日` | 最終更新日 |

**注意**: 上記プロパティは `/adr` コマンドテンプレートからの推定です。正確なスキーマは Notion MCP 接続時に `notion-fetch` で検証してください。
