実装計画(plan.md)が「強度高×距離遠×変動性高」または「強度低×距離近（Low Cohesion）×変動性高」等の危険な結合バランスを生んでいないかを、`coupling-anatomy` の3軸（統合強度×距離×変動性）とバランス判定式で検証する。

**重要**: ソースコードは一切変更しない。`./tmp/coupling-gate.md` のみ出力する（`spec-check` と同型のゲートパターン）。
**規約**: CLAUDE.md の Skills 共通規約に従う

判定基準・訳語・出力フォーマットは `coupling-anatomy` スキルを参照する（本スキルでは再宣言しない）。

## パラメーター

`$ARGUMENTS` で対象plan.mdパスを指定できる。省略時は `./tmp/plan.md` を使用する。

## ワークフロー上の位置付け

| 前工程 | 本コマンド | 後工程 |
|--------|-----------|--------|
| `/design` 計画作成（または `/spec-check`） | `/coupling-gate` 結合バランスゲート | `/review-plan` 計画レビュー or `/revise` 計画修正 |

## 実行条件

以下のファイルが存在することを確認する:

- `$ARGUMENTS`（指定がない場合は `./tmp/plan.md`）— 検証対象の実装計画

ファイルが存在しない場合:

- プロセスを即座に停止する
- どの条件が満たされていないかをユーザーに報告する
- 処理を続行しない

## 実行プロセス

### フェーズ1: 入力の読み込み

- [ ] plan.md（`$ARGUMENTS` または `./tmp/plan.md`）を読み込む

### フェーズ2: 結合関係の抽出

- [ ] plan.md 記載の結合関係を抽出する（新設モジュール間の依存、外部システム接続点等）

### フェーズ3: 3軸判定

- [ ] 各関係を `coupling-anatomy` の3軸で判定する。plan.md は意図ベースのため実コードはまだ存在しない。「plan記載に基づく推測」である旨を明記した上で判定する

### フェーズ4: レポート出力

`./tmp/coupling-gate.md` に以下を出力する:

- PASS / WARN / FAIL 判定
- 結合関係テーブル（`coupling-anatomy` の**推測ベース変種**: 想定モジュール名/概念名、根拠列は `plan.md:L{number}` または `(推測)` タグ）
- `review-plan` と共通の重要度分類（❌Critical / ⚠️Warning / ✅Good。`review-plan` が読み込んだ際にそのまま統合しやすいように）

#### 判定基準

- **PASS**: 強度高×距離遠×変動性高等の危険な結合バランスが検出されず（BALANCEがすべてtrue）、かつ各関係の根拠が `(推測)` タグのみに依存していない
- **WARN**: 軽微な懸念あり、または BALANCE はすべて true だが根拠となる記述が plan.md に見当たらず `(推測)` タグのみに依存する関係がある
- **FAIL**: 強度高×距離遠×変動性高の組み合わせが存在する（BALANCE=falseとなる関係がある）

### フェーズ5: 完了報告

- 判定結果（PASS / WARN / FAIL）
- 検出した危険な結合バランスの件数と概要
- 次のステップの案内:
  - **PASS の場合**: `/review-plan` または `/implement`
  - **WARN / FAIL の場合**: `/revise` で計画を修正
