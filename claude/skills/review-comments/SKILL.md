# PR レビューコメント対応スキル

PR 上の未解決レビューコメント（Gemini、Greptile、人間レビュアー等）を取得し、各コメントに対して「対応済み」「却下（理由付き）」「要対応」を判定した上で、**返信メッセージの投稿と resolve を実行する**。

**規約**: CLAUDE.md の Skills 共通規約に従う
## パラメーター

`<PR URL or number>` — 対象の PR を指定する。省略時は現在のブランチの PR を自動検出する。

## When to Use

- `/implement` や手動修正の後、PR 上のレビューコメントに返信・resolve したいとき
- bot（Gemini、Greptile、CodeRabbit 等）のレビュー指摘にまとめて対応したいとき
- レビュー指摘を検討し、対応/却下の判断を整理したいとき

## 実行条件

- 対象の PR が存在しアクセス可能であること
- 未解決のレビューコメントが 1 件以上存在すること

条件を満たさない場合は即座に停止し、ユーザーに報告する。

## 実行プロセス

### Phase 1: PR コメントの取得

```bash
# PR 番号の取得（URL から抽出、または現在のブランチから検出）
pr_number=$(gh pr view --json number --jq '.number' 2>/dev/null || echo "")
```

GraphQL で未解決のレビュースレッドを取得する:

```bash
gh api graphql -f query='
query($owner: String!, $repo: String!, $number: Int!) {
  repository(owner: $owner, name: $repo) {
    pullRequest(number: $number) {
      reviewThreads(first: 100) {
        nodes {
          id
          isResolved
          isOutdated
          path
          line
          startLine
          comments(first: 50) {
            nodes {
              id
              author { login }
              body
              createdAt
              url
            }
          }
        }
      }
    }
  }
}' -f owner="$owner" -f repo="$repo" -F number="$number"
```

**フィルタリング**:
- `isResolved: true` のスレッドは除外する
- **人間レビュアーのコメントは対象外**: コメント作成者の login が以下の bot パターンに該当する場合のみ処理対象とする。それ以外（人間レビュアー）はスキップし、Phase 4 の報告で「👤 人間レビュー（対象外）」として一覧表示のみ行う
  - `[bot]` を含むユーザー名（例: `gemini-code-assist[bot]`, `greptile-apps[bot]`, `github-actions[bot]`）
  - 既知の AI レビュー bot: `copilot`, `coderabbit`, `codium` 等

### Phase 2: 対応状況の分析

各未解決コメント（bot・人間とも）について、以下の情報源を照合して対応状況を判定する。
人間レビュアーのコメントは自動返信しないが、**返信レコメンドを生成するために同じ分析を行う**:

1. **現在のコード**: コメントが指摘している箇所の現在の実装を読む
2. **git diff**: コメント投稿後のコミットで該当箇所が変更されているか確認
3. **`./tmp/fix-plan.md`**（存在する場合）: 修正計画の却下判断を参照
4. **`./tmp/review/unified.md`**（存在する場合）: 自社レビュー結果を参照
5. **既存パターン**: 指摘内容がプロジェクトの慣例と矛盾していないか確認

### Phase 3: 対応方針の決定

各コメントを以下のいずれかに分類する:

| 分類 | 条件 | アクション |
|------|------|-----------|
| ✅ 対応済み | コミットで指摘箇所が修正されている | コミットハッシュと修正内容を返信し resolve |
| ❌ 却下 | 指摘がプロジェクト慣例・設計判断と矛盾する | 理由を返信し resolve |
| ⚠️ 要対応 | 未修正かつ妥当な指摘 | ユーザーに対応方針を確認 |
| ℹ️ 質問 | 指摘が質問形式で回答が必要 | 回答を返信し resolve |

### Phase 4: ユーザー確認

分析結果をテーブル形式でユーザーに提示する:

```
## レビューコメント対応方針

| # | ソース | 指摘内容 | 判定 | 返信案 |
|---|--------|---------|------|--------|
| 1 | Gemini | APIエラーハンドリング不足 | ✅ 対応済み | 9d63707 で修正... |
| 2 | Gemini | 末尾スラッシュ削除 | ❌ 却下 | 既存慣例と一致... |
| 3 | Greptile | デフォルト値上書きリスク | ✅ 対応済み | 9d63707 で修正... |

⚠️ 要対応（未修正の指摘）:
- なし

👤 人間レビュー（対象外 — 自分で返信してください）:

#### 1. @reviewer-name — 「指摘内容の要約」
- **URL**: https://github.com/.../pull/123#discussion_r...
- **指摘箇所**: `src/path/to/file.tsx:L42`
- **現在の状態**: 修正済み / 未修正 / 設計判断
- **返信レコメンド**:
  > 修正しました（コミットハッシュ）。ガード条件を強化し...
- **補足**: （返信時に考慮すべきコンテキストがあれば記載）

#### 2. @reviewer-name — 「指摘内容の要約」
...

この方針で返信・resolve してよいですか？
```

**⚠️ 要対応がある場合**: 対応方針をユーザーに確認してから次のフェーズに進む。
**全て ✅ または ❌ の場合**: ユーザーの承認を得て次のフェーズに進む。

### Phase 5: 返信の実行

各コメントに対して REST API で返信する:

```bash
gh api repos/{owner}/{repo}/pulls/{pr_number}/comments \
  -f body="$reply_message" \
  -F in_reply_to=$comment_id
```

**返信メッセージのガイドライン**:

- **言語**: 日本語で返信する（技術用語・コード例は英語のまま）
- **対応済み**: 「ご指摘ありがとうございます。」+ コミットハッシュ + 具体的な修正内容（コード例付き）
- **却下**: 「ご指摘ありがとうございます。」+ 却下理由 + 根拠（既存コードの例、公式ドキュメント等）
- **質問への回答**: 簡潔かつ具体的な回答
- **⚠️ 要対応**: 返信せず、Phase 4 でユーザーに対応方針を確認する。ユーザーが方針を決定したら、その方針に基づいて返信を作成・投稿する

返信は**並列実行**する（独立したAPIコールのため）。

**⚠️ 要対応のコメントがある場合のフロー**:
1. Phase 4 でユーザーに提示し、以下のいずれかの指示を待つ:
   - 修正する → `/fix` や手動修正の後、再度本スキルを実行
   - 却下する → 却下理由を確認し、返信・resolve を実行
   - 後で対応する → スキップして他のコメントのみ処理

### Phase 6: resolve の実行

返信が完了した全スレッドを GraphQL で resolve する:

```bash
gh api graphql -f query='
  mutation($threadId: ID!) {
    resolveReviewThread(input: {threadId: $threadId}) {
      thread { isResolved }
    }
  }' -f threadId="$thread_id"
```

resolve も**並列実行可能**だが、返信完了後に実行すること。

### Phase 7: 完了報告

ユーザーに以下を報告する:

```
## レビューコメント対応完了

| 分類 | 件数 | 返信 | resolve |
|------|------|------|---------|
| ✅ 対応済み | X件 | ✅ | ✅ |
| ❌ 却下 | Y件 | ✅ | ✅ |
| ℹ️ 質問回答 | Z件 | ✅ | ✅ |
| ⚠️ 未対応（要確認） | W件 | — | — |
| 👤 人間レビュー（対象外） | V件 | — | — |

bot コメント N 件に返信し、スレッドを resolve しました。
人間レビュアーのコメント V 件は対象外です（上記の URL と返信レコメンドを参考に、自分で返信してください）。
```

**エラーハンドリング**:
- API コールが失敗した場合、失敗したコメントの URL を報告し、手動対応を依頼する
- 部分的に成功した場合、成功分のみ報告し、失敗分は別途リストアップする
