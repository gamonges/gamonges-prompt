## レビューチェックリスト

全 Phase 完了後、以下を自己チェックする:

- [ ] ドメインイベントの消費者と副作用を明記したか
- [ ] 「動詞か名詞か」テストで Entity と Command を区別したか
- [ ] 各集約の不変条件を定義したか
- [ ] enum を設計した場合、MECE を検証したか
- [ ] 既存コードベースのパターン（ShadowIt, AppIntegration 等）と整合するか
- [ ] ユビキタス言語が Phase 2-3 の議論内容と整合するか
- [ ] コンテキスト境界を越える関心事の持ち込みがないか（例: HR-Force の概念を IT-Force に混入）
- [ ] 暗黙の除外ケース（NO_SIGNAL 等のエッジケース）を明示したか
- [ ] ReadModel で導出すべき概念と永続化すべき概念を区別したか
- [ ] 各 Phase の「合意の経緯」に検討した選択肢と却下理由が記録されているか
