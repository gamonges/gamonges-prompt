テーマ・コンテキストから、出力形式に依存しない構造化アウトラインを生成する。

**出力**: `./tmp/outline.md`
**入力**: `./tmp/context.md`（テーマ、ターゲット、制約を記述したファイル）
**規約**: CLAUDE.md の Skills 共通規約に従う
## ワークフロー上の位置付け

| 前工程 | 本スキル | 後工程 |
|--------|---------|--------|
| `/ask` 調査 | `/outline` アウトライン生成 | `/marp` スライド生成 or `/blog` ブログ生成 |

## パラメーター

`$ARGUMENTS` でコンテキストファイルのパスを指定できる。省略時は `./tmp/context.md` を使用する。

## 実行条件

以下のファイルが存在することを確認する:

- `$ARGUMENTS`（指定がない場合は `./tmp/context.md`）— テーマ・ターゲット・制約の情報
- `./tmp/research.md`（任意）— `/ask` の調査結果。存在する場合は素材として自動参照する

ファイルが存在しない場合:

- プロセスを即座に停止する
- どの条件が満たされていないかをユーザーに報告する
- 処理を続行しない

## ワークフロー

### Phase 0: 既存 outline.md の確認

`./tmp/outline.md` が既に存在する場合、**プロセスを停止**してユーザーに確認する:

- 既存の outline.md のタイトルとセクション数を表示する
- 以下の選択肢を提示する:
  1. **上書き**: 既存の outline.md を破棄し、新しいアウトラインで置き換える
  2. **キャンセル**: アウトライン作成を中止する

### Phase 1: コンテキスト読み込み

- [ ] コンテキストファイル（`$ARGUMENTS` または `./tmp/context.md`）を読み込む
- [ ] テーマ、ターゲット、制約、トーンを抽出する
- [ ] `./tmp/research.md` が存在する場合、調査結果を素材として取り込む
  - 具体的なデータ（PR一覧、技術的事実、数値）をアウトラインの「補足素材」候補として整理する

### Phase 2: 対話的深掘り

context.md の情報量に基づいて対話の要否を判定する。

**スキップ条件**: 以下の3要素がすべて context.md に含まれている場合、Phase 3 に直接進む:
1. **ターゲット**: 誰に向けたコンテンツか（例: 「クラウドネイティブに関心のあるエンジニア50名」）
2. **核心メッセージ**: 一文で表現できる伝えたいこと（例: 「技術選定はプロダクト判断である」）
3. **制約**: 時間制限または文字数（例: 「15分登壇」「3000-5000字」）

**3要素のいずれかが不足している場合**、以下を確認する（1-2往復）:
- テーマの焦点: 広いテーマのどこに絞るか
- ターゲットの解像度: 相手の前提知識・関心事
- 核心メッセージの方向性: 「結局何を持ち帰ってほしいか」

### Phase 3: アウトライン生成

- [ ] 核心メッセージを支える論理構造を設計する（導入→展開→まとめ）
- [ ] セクションに分割し、各セクションの役割を定義する
- [ ] 各セクションに以下を割り当てる:
  - ポイント: このセクションで伝えること
  - キーワード: 含めるべき技術用語・固有名詞
  - 補足素材: コード例、図、データ、エピソード（research.md から引用可）
  - 想定分量: 参考値（下流スキルはセクションの情報密度から独自に判断する）

### Phase 4: 出力・フィードバック

- [ ] `./tmp/outline.md` に以下のテンプレートで出力する
- [ ] ユーザーにアウトラインを提示する
- [ ] フィードバックがあれば修正し、再出力する

## 出力テンプレート

```markdown
# [タイトル]

## メタ情報

- **ターゲット**: [誰に向けて]
- **核心メッセージ**: [一文で]
- **制約**: [時間制限 or 文字数]
- **トーン**: [技術解説 / 体験共有 / チュートリアル 等]

## アウトライン

### 1. [セクション名]

- **ポイント**: [このセクションで伝えること]
- **キーワード**: [含めるべき技術用語・固有名詞]
- **補足素材**: [コード例、図、データ、エピソード]
- **想定分量**: [スライド2-3枚分 / 500-800字 等]（参考値）

### 2. [セクション名]

...
```

## 品質チェックリスト

出力完了後、以下を自己チェックする:

- [ ] 核心メッセージが1文で言語化されているか
- [ ] 各セクションのポイントが核心メッセージを支える論理構造になっているか
- [ ] ターゲットにとって不要な情報が混入していないか
- [ ] セクション間の流れが論理的か（導入→展開→まとめ）
- [ ] research.md の素材が適切にマッピングされているか（存在する場合）
- [ ] 出力形式固有の情報（スライド番号、CSS クラス、SEO タグ等）が混入していないか
