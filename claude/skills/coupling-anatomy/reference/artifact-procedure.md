# 結合の解剖アーティファクト作成手順

`coupling-plan-diff --artifact` 指定時のみ参照する手順。実装計画(plan.md)と実コードの差分から、結合構造の変化を可視化する HTML artifact を作成する際の実務手順（ハマりどころ含む）。

## 前提スキル

`artifact-design`、`dataviz` の2スキルを **Skill ツールで明示ロードしてから作業する**（暗黙知で代用しない）。

## 作成手順

1. `plan.md` を読み、実装の概念・モジュール一覧・意思決定を把握する
2. `git diff --stat` / `git status` で実際のコード変更を確認する。plan記載と実コードの乖離がないか、要所は `grep -n` で実クラス名・メソッド名の実在を確認してから可視化に使う（`coupling-anatomy` の事実/推測方針: plan.md は「意図」、diff は「事実」。事実を優先する）
3. モジュールをプロジェクトのアーキテクチャ層でグルーピングし、NEW/MOD/既存流用（無変更のまま新しい呼び出しを受け入れた既存資産）のバッジを付けて「見取り図」を作る（概念整理パート）
4. 結合の形パートは、主要な結合関係を **10件程度に厳選**し、それぞれ実コードの事実に基づき統合強度・距離・変動性を判定する（推測でなく呼び出し元/呼び出し先の実装を実際に読んで判断する）。「統合強度×距離」の散布図＋色/形=変動性、が最も情報量が高い形式
5. `dataviz` スキルの配色検証手順に従い、CVD安全性を確認してから使う。全 PASS でも直接ラベル必須の指示に従い、各点に直接ラベル+title を必ず付ける
6. `artifact-design` スキルをロードし、この種の成果物（技術ドキュメント/レビュー資料）の治療レベルを判断してから設計トークンを決める
7. 散布図 SVG は**固定 viewBox + 手計算座標**で組む（実行時の JS レイアウト測定に頼らない方が静的アーティファクトでは頑健）。危険域（強度高×距離遠）を矩形で網掛けし、ヒューリスティックを視覚的に表現する
8. 特筆すべき結合には番号バッジ＋テキスト callout で解説を添える。ただの散布図で終わらせず、「何を読み取ればよいか」まで書く
9. ローカルでの確認は `file://` で HTML ファイルを直接開く方式を使う（claude.ai に公開した URL は認証が必要で外部ブラウザツールから見えない場合がある）。light/dark 両方を確認する
10. 詳細テーブルには**必ず「呼び出し元→呼び出し先」の正式なクラス名をフルで書く**。チャート上の点ラベルは省略形でよいが、表や `<title>`（ホバーツールチップ）には省略しない

## ハマったポイント

- 手書き CSS のカスタムプロパティ値に typo でスペースが入り（例: `#ab a2f5`）無効な値になることがある。ファイル生成直後に怪しい値（hex なのに空白を含む等）がないか確認する
- 手計算 SVG 座標で、同じ強度帯に複数の点が集まると、点ラベル同士や番号バッジが重なりやすい。対処: (a) マーカーの x 座標をわずかにずらす (b) 片方のラベルを `text-anchor="end"` で逆方向に伸ばす (c) badge をラベル/ring/他マーカーと衝突しないオフセットに置く。スクリーンショットを撮ってから微調整するのが最短ルート
