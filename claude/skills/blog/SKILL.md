アウトラインからテックブログ記事のドラフトを生成する。

**出力**: `./tmp/blog-draft.md`
**入力**: `$ARGUMENTS` または `./tmp/outline.md`
**規約**: CLAUDE.md の Skills 共通規約に従う
## ワークフロー上の位置付け

| 前工程 | 本スキル | 後工程 |
|--------|---------|--------|
| `/outline` アウトライン生成 | `/blog` ブログ記事生成 | 手動で公開プラットフォームへ（Zenn, Qiita, note 等） |

## パラメーター

`$ARGUMENTS` で outline.md のパスを指定できる。省略時は `./tmp/outline.md` を使用する。

## 実行条件

以下のファイルが存在することを確認する:

- `$ARGUMENTS`（指定がない場合は `./tmp/outline.md`）— アウトライン
- `./tmp/research.md`（任意）— 技術的詳細の補完。存在する場合は自動参照する

ファイルが存在しない場合:

- プロセスを即座に停止する
- どの条件が満たされていないかをユーザーに報告する
- 処理を続行しない

## ワークフロー

### Phase 1: 入力読み込み

- [ ] アウトラインファイル（`$ARGUMENTS` または `./tmp/outline.md`）を読み込む
- [ ] メタ情報からトーンと分量を決定する:
  - 制約に文字数指定があればそれに従う
  - 指定がなければ 3000-5000 文字を目安とする
- [ ] `./tmp/research.md` が存在する場合、技術的詳細の補完素材として取り込む

### Phase 2: 記事構成設計

- [ ] アウトラインのセクションを記事の見出し構成に変換する:
  - H1: 記事タイトル（1つのみ）
  - H2: 大セクション（アウトラインのセクションに対応）
  - H3: 詳細（セクション内の細分化）
- [ ] リード文の位置を決定する（タイトル直後）
- [ ] まとめセクションの位置を決定する（末尾）
- [ ] コードブロック・図表の配置位置を計画する

### Phase 3: 散文化

- [ ] 各セクションを文章に展開する:
  - 箇条書きのポイントを文章として展開する
  - キーワードを文脈に自然に織り込む
  - 補足素材をコードブロック・図表として配置する
- [ ] コードブロックの前後に「なぜそうしたか」の説明を付ける
- [ ] 専門用語には初出時に説明を付加する

### Phase 4: 仕上げ・出力

- [ ] リード文を生成する（3-5文で記事全体を要約、結論ファースト）
- [ ] まとめセクションを生成する（核心メッセージの再確認 + 学びと次のステップ）
- [ ] 全体の一貫性をチェックする（トーン、文体、論理の流れ）
- [ ] `./tmp/blog-draft.md` に出力する
- [ ] ユーザーに記事の概要（文字数、セクション構成）を報告する
- [ ] フィードバックがあれば修正し、再出力する

## 記事スタイルガイドライン

### 文体

- 「です・ます」調を基本とする
- 技術用語は英語のまま使用する（例: Step Functions, Lambda, Prisma）
- 体験談のトーンでは「〜しました」「〜でした」を活用する

### リード文

- 3-5文で記事全体を要約する
- 結論ファースト: 「何をした結果どうなったか」を最初に述べる
- 読者が「この記事を読むべきか」を判断できる情報を含める

### コードブロック

- 前後に「なぜそうしたか」の説明を付ける（「何をしているか」ではなく）
- 言語指定を必ず付ける
- 長すぎるコードは要点を抜粋し、全体はリポジトリリンクで参照する

### 見出し

- 各見出しが独立して読んでも内容が推測できるようにする
- 「はじめに」「おわりに」のような汎用的な見出しは避け、具体的な内容を示す

### 分量

- context.md の制約に従う
- 指定がなければ 3000-5000 文字を目安とする

## 品質チェックリスト

出力完了後、以下を自己チェックする:

- [ ] リード文が記事全体の要約になっているか
- [ ] 各見出しが独立して読んでも内容が推測できるか
- [ ] コードブロックの前後に「なぜ」の説明があるか
- [ ] 専門用語に初出時の説明があるか
- [ ] まとめが核心メッセージを再確認しているか
- [ ] 分量が制約の範囲内か
- [ ] 文体が「です・ます」調で統一されているか
- [ ] アウトラインの全セクションが記事に反映されているか
